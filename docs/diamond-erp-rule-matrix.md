# 钻石 ERP 规则总表

## 1. 文档目的

本文件用于一次性冻结钻石 ERP 的核心执行规则。

本文件不是概念说明，而是后续改 `types`、`store`、页面动作、自动派单逻辑时直接对照的规则总表。

本文件覆盖以下规则：

1. 主状态机规则
2. 单据推进规则
3. 自动派单规则
4. 扫码交接规则
5. 异常触发规则
6. 库存变更规则
7. 关闭与归档规则
8. 权限与角色动作边界

## 2. 总体执行原则

### 2.1 总原则

- 系统主流程只允许向前推进，不允许自由回退
- 任何差异优先通过异常单冻结处理，不通过回退原单据解决
- 交接扫码确认不能替代正式业务复核
- 只有源头单据允许人工创建，过程单据优先由系统自动生成
- 深圳正式收货复核通过前，不允许进入分选主线
- 客户订单与采购批次不是同一对象，只在封包匹配与出货阶段汇合

### 2.2 单据层级

| 层级 | 单据 | 作用 |
| --- | --- | --- |
| 源头单据 | 采购批次、客户订单 | 回答业务从哪里开始 |
| 过程单据 | 在途流转单、收货单、分选单、细筛单、封包单、出货单 | 推进业务工序 |
| 控制单据 | 交接单、异常单、审计日志 | 控制推进、留痕、冻结、关闭 |

### 2.3 规则实现建议

后续代码实现建议采用三层结构：

1. `状态定义层`
2. `规则判断层`
3. `动作执行层`

不要把规则直接散落在页面里判断，避免页面越多越难维护。

## 4. 自动派单总表

## 4.1 自动派单原则

- 自动派单只在“上一步已真正完成”后触发
- 触发时必须同时满足状态条件与数据完整条件
- 自动派单失败时，系统应保留异常或待处理提示，不应静默跳过

## 4.2 自动派单规则表

| 规则ID | 触发事件 | 前置条件 | 自动生成内容 | 结果归属 |
| --- | --- | --- | --- | --- |
| AUTO-001 | 采购批次确认 | 采购批次 `confirmed` | 印度到香港待发运任务 | 采购员 |
| AUTO-002 | 印度到香港在途到达 | 在途单 `arrived` 且站点为香港 | 香港待收货任务、香港交接任务 | 香港收货员 |
| AUTO-003 | 香港收货复核通过 | 香港收货单 `reviewed` | 香港到深圳待发运任务、转深圳交接任务 | 香港仓人员 |
| AUTO-004 | 香港到深圳在途到达 | 在途单 `arrived` 且站点为深圳 | 深圳待收货任务、深圳交接任务 | 深圳收货员 |
| AUTO-005 | 深圳收货入库完成 | 深圳收货单 `posted` | 待分选任务、分选领料交接任务 | 分选员 |
| AUTO-006 | 分选复核通过 | 分选单 `reviewed` | 待细筛任务、细筛领料交接任务 | 细筛员 |
| AUTO-007 | 细筛复核通过 | 细筛单 `reviewed` | 待封包任务、封包交接任务 | 封包员 |
| AUTO-008 | 封包复核通过并入库存 | 封包单 `stored` | 可匹配库存记录 | 销售/系统 |
| AUTO-009 | 客户订单进入匹配中 | 有可匹配封包库存 | 封包匹配待办 | 销售/跟单 |
| AUTO-010 | 客户订单匹配完成 | 客户订单 `ready_to_ship` | 出货待复核任务 | 出货复核人 |
| AUTO-011 | 出货复核通过 | 出货单 `ready_to_ship` | 出货执行任务、客户签收交接任务 | 出货员 |
| AUTO-012 | 客户签收完成 | 出货交接 `completed` | 归档关闭任务 | 系统/跟单 |

## 4.3 自动派单阻断规则

以下情况下不允许生成下一步待办：

1. 当前单据状态为 `exception`
2. 当前单据仍未完成必须交接
3. 当前单据存在未关闭关键异常
4. 当前单据关键字段为空
5. 当前单据未完成复核

## 5. 单据推进前置条件总表

| 流程节点 | 推进动作 | 必须满足的前置条件 |
| --- | --- | --- |
| 采购批次 -> 在途 | 发起印度到香港在途 | 采购批次已确认、发运信息完整 |
| 在途 -> 香港收货 | 创建香港收货单 | 在途已到达香港、交接已发起 |
| 香港收货 -> 香港转深圳 | 发起转深圳在途 | 香港收货复核通过、无重大差异 |
| 香港转深圳 -> 深圳收货 | 创建深圳收货单 | 在途已到达深圳、交接已发起 |
| 深圳收货 -> 分选 | 生成待分选任务 | 深圳收货已复核通过并正式入库 |
| 分选 -> 细筛 | 生成待细筛任务 | 分选结果提交并复核通过 |
| 细筛 -> 封包 | 生成待封包任务 | 细筛结果复核通过 |
| 封包 -> 出货匹配 | 建立可匹配库存 | 封包复核通过且库存已建立 |
| 客户订单 -> 出货 | 生成出货单 | 匹配封包已满足订单要求 |
| 出货 -> 客户签收 | 发起客户签收交接 | 出货执行完成、物流信息完整 |
| 客户签收 -> 归档 | 关闭出货与客户订单 | 客户签收完成、关闭前校验通过 |

## 6. 扫码交接规则总表

## 6.1 必须生成交接单的节点

| 节点 | 是否必须交接 | 说明 |
| --- | --- | --- |
| 印度到香港到货 | 是 | 到港交接与收货不能省略 |
| 香港转深圳发运 | 是 | 跨站点转运必须留痕 |
| 深圳收货入库 | 是 | 到仓交接必须确认 |
| 分选领料 | 是 | 从库存领料必须留痕 |
| 分选交回 | 是 | 分选结果交回必须留痕 |
| 细筛交接 | 是 | 细筛到封包必须留痕 |
| 封包复核 | 是 | 封包放行前需交接 |
| 出货签收 | 是 | 客户签收是最终留痕节点 |

## 6.2 交接创建规则

| 规则ID | 创建时机 | 生成字段 | 备注 |
| --- | --- | --- | --- |
| HD-001 | 上一步完成后 | businessType、relatedNo、fromNode、toNode、应交接重量、应交接粒数 | 系统自动生成待交接 |
| HD-002 | 有已绑定企微账号 | corpWechatUserId、corpWechatDisplayName | 优先自动带出 |
| HD-003 | 未绑定企微但规则强制 | 生成交接待办但标记高风险 | 后续若未补齐应触发异常 |

## 6.3 交接放行规则

| 节点 | 放行条件 |
| --- | --- |
| 到港交接 | 扫码完成且收货已创建 |
| 转深圳交接 | 香港收货复核通过 |
| 深圳入库交接 | 到仓交接完成且深圳收货复核通过 |
| 分选领料交接 | 深圳可分选库存充足 |
| 分选交回交接 | 分选结果完整 |
| 细筛交接 | 细筛结果完整 |
| 封包复核交接 | 封包数据与封签码完整 |
| 客户签收交接 | 客户签收结果返回或人工确认完成 |

## 6.4 交接超时规则

| 场景 | 超时条件 | 系统动作 |
| --- | --- | --- |
| 待扫码未处理 | 超过配置时长 | 交接单变为 `timeout`，自动挂异常 |
| 已扫码未确认 | 超过配置时长 | 交接单变为 `timeout`，提醒责任人 |
| 客户签收未返回 | 超过配置时长 | 出货异常自动生成 |

## 7.2 异常触发后的系统动作

| 动作 | 说明 |
| --- | --- |
| 生成异常单 | 建立正式异常记录 |
| 冻结关联业务单 | 相关单据状态改为 `exception` 或冻结态 |
| 阻断自动派单 | 不再生成后续待办 |
| 增加仪表盘与菜单提醒 | 全局显示异常数量 |
| 记录审计日志 | 留存触发原因和时间 |

## 7.3 异常关闭前校验总表

| 异常类型 | 关闭前必须满足 |
| --- | --- |
| 在途异常 | 在途已到达或签收，运输问题已解除 |
| 收货异常 | 差异已复核并有明确处理结果，允许入库或明确冻结归档 |
| 分选异常 | 分选结果已复核通过或已重新处理 |
| 细筛异常 | 复检通过或有最终等级确认 |
| 封包异常 | 封签、数量、复核结果已补齐 |
| 出货异常 | 出货单已恢复可签收或已重新完成交接 |
| 扫码交接异常 | 交接单已完成闭环；若强制企微实名，则实名字段完整 |

## 8. 库存变更规则总表

## 8.1 库存阶段定义

| 库存阶段 | 含义 |
| --- | --- |
| 在途库存 | 正在跨站点运输 |
| 香港中转在库 | 香港已收货但未转深圳 |
| 深圳待入库 | 已到深圳待复核 |
| 深圳可分选库存 | 深圳入库复核通过，可进入分选 |
| 生产中库存 | 已领料进入分选/细筛/封包 |
| 可出货库存 | 封包复核通过，可匹配客户订单 |
| 冻结库存 | 挂异常、不可继续使用 |
| 已出货库存 | 已执行出货，待客户签收 |
| 已完成库存 | 客户已签收，流程归档 |

## 8.2 库存变更触发规则

| 规则ID | 触发事件 | 库存变化 |
| --- | --- | --- |
| INV-001 | 印度发运 | 转为在途库存 |
| INV-002 | 香港收货复核通过 | 转为香港中转在库 |
| INV-003 | 香港发往深圳 | 从香港中转在库转为在途库存 |
| INV-004 | 深圳收货复核通过并入库 | 转为深圳可分选库存 |
| INV-005 | 分选领料 | 从深圳可分选库存转为生产中库存 |
| INV-006 | 封包复核通过 | 转为可出货库存 |
| INV-007 | 出货执行 | 从可出货库存转为已出货库存 |
| INV-008 | 客户签收完成 | 转为已完成库存 |
| INV-009 | 任意节点挂异常 | 转为冻结库存 |

## 9. 关闭与归档规则总表

## 9.1 单据关闭规则

| 单据 | 关闭条件 |
| --- | --- |
| 采购批次 | 关联业务链已完成，库存已归档，无未关闭异常 |

| 收货单 | 已完成复核并已入库，或已异常转出 |

| 封包单 | 已入成品库存或异常关闭 |
| 出货单 | 客户已签收并归档，或异常关闭 |

| 异常单 | 关闭前校验通过且已补齐关闭说明 |

## 10. 角色动作边界总表

| 角色 | 允许执行的关键动作 | 不允许执行的关键动作 |
| --- | --- | --- |
| 采购员 | 创建采购批次、发起首段在途 | 关闭异常、越权入库 |
| 采购主管 | 确认采购批次、审核关键采购动作 | 代替收货员录入实收 |
| 香港收货员 | 香港收货、提交差异说明 | 直接放行重大差异 |
| 深圳收货员 | 深圳收货、提交入库申请 | 绕过复核直接入库 |
| 复核人 | 放行、转异常、关闭异常 | 直接修改源头单据核心来源字段 |
| 分选员 | 领料、作业、提交结果 | 关闭收货异常 |
| 细筛员 | 领料、作业、提交结果 | 越过封包复核直接放行出货 |
| 封包员 | 封包、录入封签 | 直接建立客户订单 |
| 销售/跟单 | 创建客户订单、匹配封包、申请出货 | 直接改生产结果 |
| 出货员 | 发货、跟进签收、补录物流 | 跳过客户订单直接闭环归档 |
| 系统管理员 | 配置编号、阈值、账户、企业微信参数 | 代替业务角色处理实际业务数据 |
